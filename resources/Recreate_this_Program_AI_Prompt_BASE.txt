PART 2: PERCEPTRON ALGORITHM
================================================================================

The perceptron is a single-layer neural network that classifies binary patterns.

FORMULA:
  OUTPUT = SUM( Switch(i) * Weight(i) ) + Bias

Where:
  - Switch(i) = +1 if ON, -1 if OFF (not 0/1!)
  - Weight(i) = dial value from -30 to +30
  - Bias = bias dial value from -30 to +30
  - OUTPUT = real number (displayed on analog meter)

CLASSIFICATION:
  - Positive output = one class (e.g., "T" shape)
  - Negative output = other class (e.g., "J" shape)

LEARNING RULE (Perceptron Learning Algorithm):
When pressing [Learn +] (want POSITIVE output) AND output is currently wrong:
  - For each ON switch: weight[i] += learningRate
  - For each OFF switch: weight[i] -= learningRate
  - Bias += learningRate

When pressing [Learn -] (want NEGATIVE output) AND output is currently wrong:
  - For each ON switch: weight[i] -= learningRate
  - For each OFF switch: weight[i] += learningRate
  - Bias -= learningRate

IMPORTANT: If output is already correct sign, NO changes are made.
All weights clamped to range [-30, +30].

PART 2: PERCEPTRON AS ANALOG CIRCUIT (ELECTRICAL FOUNDATION)
================================================================================

The perceptron is implemented as an INVERTING SUMMING AMPLIFIER using an LM358 op-amp.

**MATHEMATICAL OPERATION:**
```
OUTPUT = -Rf * Σ(INPUT[i] / Rin[i])
       = -Rf * Σ(INPUT[i] * weight[i] / Rf)
       = -Σ(INPUT[i] * weight[i])
```

Where: Rin[i] = Rf / |weight[i]|

**CIRCUIT TOPOLOGY (Required Components):**

1. **Input Stage** (per input):
   - SPDT switch: Selects +5V or GND
   - Input resistor Rin: Value = 10kΩ / |weight|
   - Positive weights require SWAPPED switch wiring (to compensate for op-amp inversion)

2. **Summing Junction**:
   - All input resistors connect to one node
   - This node connects to op-amp Pin 2 (inverting input)

3. **Op-Amp (LM358 Section A - Inverting Summer)**:
   - Pin 2 (IN-): Summing junction
   - Pin 3 (IN+): Ground or virtual ground (2.5V for single supply)
   - Pin 1 (OUT): Output
   - Pin 8 (V+): +5V power
   - Pin 4 (GND): Ground
   - Gain = -Rf (negative due to inverting configuration)

4. **Feedback Resistor**:
   - Rf = 10kΩ (reference resistor)
   - Connects Pin 1 (output) back to Pin 2 (inverting input)
   - Sets closed-loop gain

5. **Output Stage**:
   - Current limiting resistor (470Ω)
   - LED (green, 5mm)
   - Lights when output is positive

6. **Power Supply**:
   - 9V battery
   - LM7805 voltage regulator (+5V output)
   - Decoupling capacitors (10µF, 0.1µF)

**FORMULA VERIFICATION:**
With switch representing +1 (ON) or -1 (OFF):
```
Vout = -Rf * Σ(Switch[i] * |weight[i]| / Rf)
     = -Σ(Switch[i] * |weight[i]|)
```
With swapped wiring for positive weights, effective input sign flips, producing:
```
OUTPUT = -Σ(Switch[i] * weight[i])
```

PART 11: KEY IMPLEMENTATION DETAILS & LESSONS LEARNED
================================================================================

**Critical Bugs to Avoid:**

1. **Coordinate System Consistency**:
   - Breadboard drawing uses (x, y, width, height) parameters
   - Power rail calculations must use SAME coordinate system
   - GetBottomPowerRailY(bbTop, bbHeight) must return bbTop + bbHeight - 25
   - Don't mix different coordinate origins!

2. **Component Visual vs Electrical Position**:
   - Where you DRAW a component must match where you REGISTER it for clicks
   - RegisterComponent rectangle must EXACTLY match visual bounds
   - Example: Resistor drawn from x to x+40 → Rectangle(x, y-5, 40, 10)

3. **Resistor Scaling**:
   - Resistors span 2 columns (2 × holeSpacing = 40px)
   - Don't stretch resistors to fill large gaps - use jumper wires
   - Visual component size separate from electrical span

4. **Power Rail Holes**:
   - Clusters of 5 holes with 1-space gaps (not 7-space gaps)
   - Dense distribution allows flexible component placement
   - Calculate rail hole positions accurately (GetPowerRailHole)

5. **Avoid Duplicate Code**:
   - Don't have OLD visual-only AND NEW hole-based methods
   - Remove dead code immediately after replacement
   - One source of truth for each feature

6. **Hit Testing**:
   - Don't register breadboard itself as clickable (covers everything)
   - Enlarge hit regions to match 2× scaled layout
   - Use click selection (not hover) for deliberate component inspection

7. **Debug First, Claim Later**:
   - Add visual debug markers to verify calculations
   - Don't claim fixes work until tested
   - Debug mode should show hit rectangles, coordinates, connections
