PART 2: PERCEPTRON ALGORITHM (7 MATH RULES)
================================================================================

The perceptron processes binary switch patterns through weighted connections.
Switches output +1 (ON) or -1 (OFF), never 0. Weights range -30 to +30.

CORE FORMULA (Classic 1958):
  OUTPUT = SUM( Switch(i) * Weight(i) ) + Bias

CLASSIFICATION:
  Positive output = one class (e.g., "T" shape)
  Negative output = other class (e.g., "J" shape)

--- MATH RULES (selected via Math Dial) ---

1. PERCEPTRON_CLASSIC (1 o'clock, "1958"):
   1-to-1 connectivity: each input connects to one hidden node.
   hidden[j] = inputs[j] * weights[j]
   OUTPUT = SUM(hidden) + bias
   Learning: Update only when wrong (original perceptron rule).

2. RULE_1958_SUM (2 o'clock, "1958+"):
   Fully connected: ALL inputs connect to each hidden node.
   hidden[j] = SUM_i(inputs[i] * weights[j])
   OUTPUT = SUM(hidden) + bias
   Learning: Same as classic (update only when wrong).

3. RULE_1958_AVG (3 o'clock, "1958m"):
   Fully connected, averaged.
   hidden[j] = (1/N) * SUM_i(inputs[i] * weights[j])
   OUTPUT = SUM(hidden) + bias

4. RULE_1958_DIV_SUM (4 o'clock, "1958/+"):
   Divide inputs by N first, then sum.
   hidden[j] = SUM_i((inputs[i]/N) * weights[j])
   OUTPUT = SUM(hidden) + bias

5. RULE_1958_DIV_AVG (5 o'clock, "1958/m"):
   Divide inputs AND average output.
   hidden[j] = (1/N) * SUM_i((inputs[i]/N) * weights[j])
   OUTPUT = SUM(hidden) + bias

6. WIDROW_HOFF (9 o'clock, "1960"):
   1-to-1 connectivity (same topology as classic).
   Same calculation as classic perceptron.
   Learning: Continuous error, MSE gradient descent.
   w(t+1) = w(t) + eta * (desired - actual) * input
   Uses 0.1x learning rate scaling. Updates even when "close" to correct.

7. BACKPROP (10 o'clock, "1986"):
   Full multi-layer network with learned hidden weights.
   Uses separate HiddenWeights[inputCount, hiddenCount], HiddenBiases[],
   OutputWeights[] arrays (not the knob weights for hidden layer).
   Forward: z[j] = SUM(HiddenWeights[j,i] * input[i]) + HiddenBiases[j]
            h[j] = ReLU(z[j]) = max(0, z[j])
            output = SUM(OutputWeights[j] * h[j]) + bias
   Backward: Backpropagation through hidden layer.
   Uses 0.01x learning rate scaling.

Positions 6, 7, 8 o'clock are empty (reserved for future expansion).

--- LEARNING RULES ---

1958 variants (rules 1-5): Update only when classification is WRONG.
  Learn+ (want positive): For ON switches weight += rate; OFF switches weight -= rate; bias += rate
  Learn- (want negative): For ON switches weight -= rate; OFF switches weight += rate; bias -= rate

1960 (Widrow-Hoff): Continuous error update every time.
  w += learningRate * 0.1 * error * input (where error = desired - actual)

1986 (Backprop): Full backpropagation through hidden layer.
  Uses learningRate * 0.01 scaling. Updates HiddenWeights, HiddenBiases,
  OutputWeights, and output Bias.

All weights clamped to [-30, +30] after every update.
Default learning rate: 10.0.

PART 5: CUSTOM CONTROLS SPECIFICATIONS
================================================================================

All controls use GDI+ with SmoothingMode.AntiAlias and double buffering.
All inherit from Control with ControlStyles for flicker-free rendering.

----- SwitchControl -----
Default: 50x80 (scalable for larger grids)
Components:
  - Red LED at top (glow when ON, dark when OFF)
  - Toggle track (dark gray rounded rectangle)
  - Toggle handle (slides up=ON, down=OFF)
  - ON color: Color.FromArgb(100, 100, 100)
  - OFF color: Color.FromArgb(70, 70, 70)
Behavior:
  - Click or Space toggles state
  - StateChanged event
  - Value: +1 when ON, -1 when OFF

----- KnobControl -----
Default: 70x90 (scalable)
Range: -30 to +30, Step: 0.05
Components:
  - Circular knob with metallic gradient
  - Pointer indicator line
  - Tick marks (major at -30, 0, +30)
  - Value display below knob (format "0.00")
  - Zero points straight UP
Behavior:
  - Click to select (becomes active knob)
  - Arrow keys change value when active (arrows must NOT tab between controls)
  - Mouse drag rotates knob
  - Mouse wheel changes value
  - ValueChanged event

----- SettingsKnobControl -----
Configurable-range variant of KnobControl.
Properties: MinValue, MaxValue, Step, TickValues, MajorTickValues,
  MinValuePointsUp, ValueFormat
Special: BelowMinimumAttempted event (fires when user drags below min)
Used for: Grid Size (1-10), Learning Rate (-30 to +30)
Features auto-repeat timer for smooth keyboard adjustment (KnobClick).

----- ConfigKnob (Math Dial) -----
Discrete-position rotary dial for selecting math rules.
Size: 120x78
Positions (clock face):
  1 o'clock: "1958" (PERCEPTRON_CLASSIC) - default
  2 o'clock: "1958+" (RULE_1958_SUM)
  3 o'clock: "1958m" (RULE_1958_AVG)
  4 o'clock: "1958/+" (RULE_1958_DIV_SUM)
  5 o'clock: "1958/m" (RULE_1958_DIV_AVG)
  6-8 o'clock: empty (show small dots)
  9 o'clock: "1960" (WIDROW_HOFF)
  10 o'clock: "1986" (BACKPROP)
Contains MathRule enum used by PerceptronEngine.
Behavior: Click+drag, arrow keys, mouse wheel to switch positions.
Event: RuleChanged fires when position changes.
Labels rendered in Consolas 7pt around the dial circumference.
Active position tick is brighter; inactive ticks are dimmer.

----- AnalogMeterControl -----
Size: 180x130, Range: -100 to +100
  - Circular arc gauge with graduation marks
  - Animated needle (16ms timer for smooth movement)
  - Metallic bezel, dark face, light markings
  - No unit label on face

----- OutputLedControl -----
Large green LED with PathGradientBrush glow effect when ON.
"OUTPUT" label below.

----- MechanicalPushButton -----
Two variants: IsSquare (rectangular) or round (circular).
Size: ~45-50x55
Components:
  - Metal plate label above with text and corner screw divots
  - Recessed housing, bezel gradient, button face gradient
  - Glow effect using PathGradientBrush (color varies by button)
  - Visual press animation on mouse down
Glow colors:
  - BRAIN, MAN.: Yellow Color.FromArgb(255, 220, 80)
  - MAKE: Green Color.FromArgb(100, 180, 100)
  - SAVE: Green only when weights dirty, else none
  - LOAD: Faint red Color.FromArgb(200, 80, 60)
  - LEARN+: Green when any switch ON AND output <= 0
  - LEARN-: Yellow when any switch ON AND output > 0
  - RESET: Red when any switch ON or any knob non-zero
Event: ButtonClick

----- ArrowButton -----
Triangular d-pad buttons. Directions: Up, Down, Left, Right.
Up/Down: 40x16, Left/Right: 16x40
Pale yellow glow Color.FromArgb(255, 220, 80).
Clicking shifts entire switch pattern in that direction.
Bits that shift off the edge disappear (no wrap).

----- MetalLabelControl -----
Engraved metal plate with text.
Gradient: Color.FromArgb(70,75,70) to Color.FromArgb(50,55,50)
Four corner screw divots. Beveled edges.
Text: Color.FromArgb(130,135,130), shadow Color.FromArgb(30,35,30)
Properties: LabelText, Subdued (muted colors), CustomTextColor

----- MetalPlateControl -----
Larger instruction plate, same metallic styling.
Property: InstructionLines (string array for multi-line text).

----- FormulaPlateControl -----
Dynamic formula display plate. Updates text based on active math rule.
Shows two lines: "OUTPUT =" and the formula for the selected rule.
  PERCEPTRON_CLASSIC: "SUM(Switch x Weight) + Bias"
  RULE_1958_SUM: "SUM(all Switch x Weight) + Bias"
  RULE_1958_AVG: "AVG(all Switch x Weight) + Bias"
  RULE_1958_DIV_SUM: "SUM(Switch/N x Weight) + Bias"
  RULE_1958_DIV_AVG: "AVG(Switch/N x Weight) + Bias"
  WIDROW_HOFF: "SUM(Switch x Weight) + Bias"
  BACKPROP: "ReLU(W1*x + b1) -> W2*h + b2"

----- TogglePlateControl -----
1950s wall-mount style rocker switch on a metal plate.
Default size: 50x75
Properties: LabelText (rendered vertically), IsToggled, ActiveColor,
  InactiveColor
Components:
  - Rounded rectangle metal plate with dark metallic gradient
  - Two mounting divots (top and bottom, centered)
  - Recessed slot with shadow for 3D depth
  - Rocker paddle that moves up (toggled) or down (not toggled)
  - Vertical text engraved on plate
Event: ToggleChanged
Used for: SETTINGS (collapses control panel), TELETYPE (placeholder)

PART 6: DYNAMIC SCALING
================================================================================

When grid size changes (1-10), switches and knobs scale down for larger grids.

Switch scaling (7x7 and larger):
  width = Math.Max(28, 50 - (gridSize - 6) * 7)
  height = Math.Max(38, 80 - (gridSize - 6) * 12)

Knob scaling (7x7 and larger):
  width = Math.Max(45, 70 - (gridSize - 6) * 8)
  height = Math.Max(58, 90 - (gridSize - 6) * 10)

At 8x8+: Knob panel gets 50% width (up from 40%). Knobs shift right
by half a knob width for better fit.

Window auto-grows to fit grid content (AutoGrowWindow method).
Prevents going off-screen by repositioning if needed.

================================================================================
PART 7: LINEAR MODE (EASTER EGG)
================================================================================

When at grid size 1 and the user drags the knob further down:
  - BelowMinimumAttempted event fires
  - Enter linear mode: _isLinearMode = true, _linearNodeCount = 2
  - Each subsequent downward drag increments _linearNodeCount (max 25)
  - Nodes wrap at 5 per row (e.g., 7 nodes = row of 5 + row of 2)
  - Allows testing non-square node counts (2, 3, 4... 25)

When user drags grid size back up from linear mode:
  - Exit linear mode, revert to normal NxN grid

PART 9: EVENT WIRING
================================================================================

Switch toggled → UpdateOutput() → recalculate, update meter/LED/debug
Knob changed → engine.SetWeight() → UpdateOutput()
Bias knob → engine.Bias → UpdateOutput()
Grid size knob → engine.Resize() → CreateGrid() → AutoGrow → UpdateOutput()
Math dial → engine.MathRule → UpdateFormulaPlate() → UpdateOutput()
Learn+ click → engine.Learn(inputs, true) → sync knobs → UpdateOutput()
Learn- click → engine.Learn(inputs, false) → sync knobs → UpdateOutput()
Reset click → all switches OFF, all knobs to 0, bias to 0
Arrow click → ShiftPattern(dx, dy) → move switch pattern → UpdateOutput()

PART 12: KEYBOARD AND MOUSE INTERACTION
================================================================================

KNOB INTERACTION:
  - Click to select (makes it the "active" knob)
  - Arrow keys change active knob value by step
  - CRITICAL: Arrow keys must NOT tab between controls
    (override PreProcessMessage to capture arrow keys)
  - Tab key moves between controls normally
  - Mouse drag rotates knob (vertical drag = value change)
  - Mouse wheel changes value

SWITCH INTERACTION:
  - Click toggles switch state
  - Space bar toggles focused switch

RESET BUTTON:
  - Turns OFF all switches and zeroes all knobs and bias

PATTERN SHIFT (D-PAD):
  - Arrow buttons shift entire switch pattern in that direction
  - Bits that shift off edge disappear (no wrap)
  - Works in both grid mode and linear mode (5-column wrap)

CENTER TOGGLE (D-PAD CENTER):
  - Cycles: Off → All ON (yellow) → All OFF → repeat
  - If some switches are on, turns all on first

================================================================================
PART 13: PERCEPTRON ENGINE
================================================================================

PerceptronEngine class manages all neural network state and computation.

Properties:
  GridSize, Weights[] (double), Bias (double), LearningRate (double),
  MathRule (ConfigKnob.MathRule),
  HiddenWeights[,] (for backprop), HiddenBiases[] (for backprop),
  OutputWeights[] (for backprop), HiddenOutputs[] (cached activations),
  IsFullyConnected (bool), UsesBackprop (bool)

Key methods:
  CalculateOutput(int[] inputs) → dispatches to correct math rule
  Learn(int[] inputs, bool desiredPositive) → dispatches to correct learning
  Resize(int newNodeCount) → rebuilds all weight arrays
  ResetWeights() → zeroes everything
  SetWeight(int index, double value) → clamp to [-30, 30]

Engine state syncs bidirectionally with UI knobs:
  Knob change → engine.SetWeight()
  Learn button → engine updates weights → knobs sync from engine

PART 14: IMPLEMENTATION NOTES
================================================================================

1. Use extension methods FillRoundedRectangle / DrawRoundedRectangle for
   Graphics (create a static GraphicsExtensions class).

2. Main form resize: WndProc intercepts WM_NCHITTEST (0x84) and returns
   HTLEFT/HTRIGHT/HTTOP/HTBOTTOM/corner codes for edge resize.

3. Grid Size knob: MinValuePointsUp = true (1 points up, increases clockwise)

4. Learning Rate knob: MinValuePointsUp = false (0 points up for symmetric
   -30 to +30 range). Default value is 10.0.

5. Switches output +1 or -1, never 0.

6. Meter output clamped to [-100, +100] for display. LED on if output > 0.

7. Control panel Resize event triggers CenterActionButtons() to reposition
   all bottom-panel controls dynamically.

8. Formula plate updates whenever math dial changes position.

9. All controls should have no tab stops for arrow keys. Override
   PreProcessMessage in KnobControl and SettingsKnobControl to capture
   arrow keys and prevent focus movement.

10. The ConfigKnob (Math Dial) positions use clock-hour mapping:
    Position 0 → 1 o'clock (30°), Position 1 → 2 o'clock (60°), etc.
    Angle formula: hourAngle = position_hour * 30° (from 12 o'clock)
