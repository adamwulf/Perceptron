PART 2: PERCEPTRON AS ANALOG CIRCUIT (ELECTRICAL FOUNDATION)
================================================================================

The perceptron is implemented as an INVERTING SUMMING AMPLIFIER using an LM358 op-amp.

**MATHEMATICAL OPERATION:**
```
OUTPUT = -Rf * Σ(INPUT[i] / Rin[i])
       = -Rf * Σ(INPUT[i] * weight[i] / Rf)
       = -Σ(INPUT[i] * weight[i])
```

Where: Rin[i] = Rf / |weight[i]|

**CIRCUIT TOPOLOGY (Required Components):**

1. **Input Stage** (per input):
   - SPDT switch: Selects +5V or GND
   - Input resistor Rin: Value = 10kΩ / |weight|
   - Positive weights require SWAPPED switch wiring (to compensate for op-amp inversion)

2. **Summing Junction**:
   - All input resistors connect to one node
   - This node connects to op-amp Pin 2 (inverting input)

3. **Op-Amp (LM358 Section A - Inverting Summer)**:
   - Pin 2 (IN-): Summing junction
   - Pin 3 (IN+): Ground or virtual ground (2.5V for single supply)
   - Pin 1 (OUT): Output
   - Pin 8 (V+): +5V power
   - Pin 4 (GND): Ground
   - Gain = -Rf (negative due to inverting configuration)

4. **Feedback Resistor**:
   - Rf = 10kΩ (reference resistor)
   - Connects Pin 1 (output) back to Pin 2 (inverting input)
   - Sets closed-loop gain

5. **Output Stage**:
   - Current limiting resistor (470Ω)
   - LED (green, 5mm)
   - Lights when output is positive

6. **Power Supply**:
   - 9V battery
   - LM7805 voltage regulator (+5V output)
   - Decoupling capacitors (10µF, 0.1µF)

**FORMULA VERIFICATION:**
With switch representing +1 (ON) or -1 (OFF):
```
Vout = -Rf * Σ(Switch[i] * |weight[i]| / Rf)
     = -Σ(Switch[i] * |weight[i]|)
```
With swapped wiring for positive weights, effective input sign flips, producing:
```
OUTPUT = -Σ(Switch[i] * weight[i])
```

PART 5: BREADBOARD - ELECTRICALLY ACCURATE LAYOUT ENGINE
================================================================================

**POCBreadboardDialog.cs - Hole-Based Circuit Layout**

**CRITICAL PRINCIPLES:**

1. **Electrical Accuracy First**: This is NOT just a visual diagram. The breadboard must
   represent a REAL circuit that can be built and will function.

2. **Hole-Based Placement**: Every component must be placed at specific (row, column)
   coordinates. Holes represent actual breadboard electrical connectivity:
   - Vertical columns of 5 holes are electrically connected (same node)
   - Holes across the center channel are NOT connected
   - Power rails run horizontally (all holes in a rail are connected)

3. **Component Positioning**:
   - Track exact hole positions for every component lead
   - Calculate hit regions that EXACTLY match visual component positions
   - Use `GetHolePos(bbLeft, bbTop, row, col)` for all coordinate calculations
   - Hole spacing: 20px (2× standard for detailed view)

4. **Electrical Connectivity**:
   - Model which holes are connected (vertical groups of 5)
   - Draw jumper wires only between different electrical nodes
   - All power connections must reach actual power rail holes
   - All ground connections must reach actual ground rail holes

**LAYOUT SPECIFICATIONS:**

**Show 4 inputs in detail** (scalable to 3-6):
- Row spacing: 4 rows per input (allows room for components and wiring)
- Start at row D (row 3) - gives room for power connections above
- Column allocations:
  - Column 2: Shared ground bus for all switches
  - Column 3: Switch +5V terminals
  - Column 5: Switch common terminals
  - Column 15-17: Input resistors (span 2 columns)
  - Column 25: Summing bus (yellow vertical wire)
  - Column 30: Op-amp (DIP-8 straddling center channel)
  - Column 40-42: Output LED resistor (span 2 columns)
  - Column 46: Output LED

**Power Rails:**
- Top and bottom rails with clustered holes
- Clusters of 5 holes with 1-space gaps (dense power distribution)
- Row/column numbering: A-Z rows, 5/10/15/20... columns

**Component Registration:**
- RegisterComponent with exact hit rectangles matching visual positions
- Rectangle must cover where component is ACTUALLY drawn
- Example: Resistor spans x to x+40, y-5 to y+5 → Rectangle(x, y-5, 40, 10)

**Wire Drawing:**
- All wires 4-5px thick for visibility
- Color-coded by input (red, blue, yellow, green, etc.)
- Ground bus: 2px (thinner to reduce visual weight)
- Straight jumper wires (not curved - simpler and clearer)

**Power Supply Selector:**
- ComboBox dropdown with options:
  - "Off-board (9V battery + 7805)" - Default, clean layout
  - "On-board (battery + regulator on breadboard)" - Shows schematic overlay
  - "USB 5V direct (no regulator)"
  - "Bench supply (wire to rails only)"

**Debug Toggle:**
- CheckBox labeled "Debug"
- When ON: Show cyan hit rectangles around all components with coordinates
- Helps verify click detection and component positioning

**Click Selection (Not Hover):**
- MouseClick event handler (not MouseMove)
- Hit test to find component at click location
- Display selected component in detail panel
- Highlight selected with yellow 3px border

**ELECTRICAL CONNECTIONS (Must All Be Present):**

Per Input:
- Red wire: Switch +5V terminal → top +5V rail
- Black wire: Switch GND terminal → shared ground bus (column 2) → top GND rail
- Colored wire: Switch common → resistor input
- Colored wire: Resistor output → summing bus (column 25)

Op-Amp:
- Red wire: Pin 8 (V+) → top +5V rail
- Black wire: Pin 4 (GND) → bottom GND rail
- Black wire: Pin 3 (IN+) → bottom GND rail (for inverting config)

Summing Bus:
- Yellow vertical wire: Collects all resistor outputs
- Yellow jumper: Summing bus → op-amp Pin 2 (IN-)
- Feedback: Pin 2 → Rf (10kΩ, 2-column resistor) → Pin 1

Output:
- Green wire: Op-amp Pin 1 → current limiting resistor (470Ω)
- Green wire: Resistor → LED anode
- Black wire: LED cathode → bottom GND rail

**VALIDATION CHECKLIST:**
- All components have power (switches, op-amp powered)
- All grounds connected to ground rails
- Summing bus collects ALL inputs
- Feedback path Pin 1 → Pin 2 complete
- No wires going off-screen or to nowhere
- Circuit path traceable from input to output
- Matches validated schematic topology

PART 11: KEY IMPLEMENTATION DETAILS & LESSONS LEARNED
================================================================================

**Critical Bugs to Avoid:**

1. **Coordinate System Consistency**:
   - Breadboard drawing uses (x, y, width, height) parameters
   - Power rail calculations must use SAME coordinate system
   - GetBottomPowerRailY(bbTop, bbHeight) must return bbTop + bbHeight - 25
   - Don't mix different coordinate origins!

2. **Component Visual vs Electrical Position**:
   - Where you DRAW a component must match where you REGISTER it for clicks
   - RegisterComponent rectangle must EXACTLY match visual bounds
   - Example: Resistor drawn from x to x+40 → Rectangle(x, y-5, 40, 10)

3. **Resistor Scaling**:
   - Resistors span 2 columns (2 × holeSpacing = 40px)
   - Don't stretch resistors to fill large gaps - use jumper wires
   - Visual component size separate from electrical span

4. **Power Rail Holes**:
   - Clusters of 5 holes with 1-space gaps (not 7-space gaps)
   - Dense distribution allows flexible component placement
   - Calculate rail hole positions accurately (GetPowerRailHole)

5. **Avoid Duplicate Code**:
   - Don't have OLD visual-only AND NEW hole-based methods
   - Remove dead code immediately after replacement
   - One source of truth for each feature

6. **Hit Testing**:
   - Don't register breadboard itself as clickable (covers everything)
   - Enlarge hit regions to match 2× scaled layout
   - Use click selection (not hover) for deliberate component inspection

7. **Debug First, Claim Later**:
   - Add visual debug markers to verify calculations
   - Don't claim fixes work until tested
   - Debug mode should show hit rectangles, coordinates, connections
